AlgoVision 2.0

Five modules, five core data structures, one shared dataset of orders moving through ingestion, cataloging, search, optimization, and routing. Everything below runs in your browser — no backend, no build step.

Queue + Hashing Arrays + Binary Search BST + Min-Heap Greedy + 0/1 Knapsack DP Weighted Graph + Dijkstra

Data ingestion

Queue · HashMap

Orders arrive and get pushed onto a queue, so they're always processed in the order they came in. A hash index sits alongside it, so any order can still be pulled up instantly by ID without scanning the line.

03

Search engine

Binary search tree · Min-heap

Two structures, two jobs. The BST keeps products ordered by price so range queries ("everything between ₹3000 and ₹10000") are fast. The heap doesn't care about order at all — it only ever needs to answer "what's most urgent right now."

04

Optimizer

Greedy · 0/1 Knapsack DP

Same orders, same budget, two strategies. Greedy grabs the cheapest orders first and stops when the budget runs out — fast, but not always the best combination. The DP knapsack checks every viable combination implicitly and always finds the one with the highest total value.
